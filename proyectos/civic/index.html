El reto

Un ERP y una intranet que no se hablaban

Civic gestiona su información en Holded, pero su intranet vivía aparte. Holded no expone una API oficial para lo que necesitaban, así que la sincronización se hacía a mano: alguien copiaba datos de un sistema a otro.

El resultado era el previsible: información desactualizada, trabajo repetitivo y margen de error en cada volcado. Necesitaban que los dos sistemas estuvieran al día sin depender de que nadie se acordara de actualizarlos.

Qué hicimos

Un plugin a medida que sincroniza solo

Construimos un plugin de WordPress a medida que lleva los datos de Holded a su intranet de forma desatendida. Como no había API oficial, automatizamos la extracción con Playwright y la orquestamos con GitHub Actions, de modo que la sincronización se ejecuta sola en segundo plano.

Sincronización Holded → intranet

Playwright + GitHub Actions, sin API oficial y sin intervención manual.

33 custom fields mapeados

Cada dato de Holded en su sitio dentro de la intranet.

Buscador multicampo sin acentos

Encontrar es inmediato, escribas como escribas el nombre.

Espejo a Google Calendar

Lo relevante se refleja en su calendario automáticamente.

El resultado

Sincronización desatendida, en producción

El plugin está en producción en su versión v1.7.3: los datos viajan solos de Holded a la intranet, sin volcados manuales y sin la información envejeciendo entre actualizaciones.

Civic es hoy un cliente recurrente: seguimos manteniendo y evolucionando el plugin versión a versión.
